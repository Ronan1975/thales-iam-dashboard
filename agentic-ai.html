Agent Identity Management

Every AI agent needs a verifiable identity with scoped permissions, session management, and revocation. 78% of orgs lack formal policies for creating or removing AI identities. Only 23% have an enterprise-wide strategy. Sources: Okta Survey; Strata Research

Consent-Aware AI Interactions

When AI agents interact with customers, CIAM must verify the agent’s authority AND the customer’s consent. A new consent layer OneWelcome can own — GDPR-compliant agent-to-human verification. No competitor offers standalone consent for AI.

Delegation Chains

Agent A calls Agent B calls Agent C — each with different permissions. Transitive trust across multi-agent systems is unsolved. Deutsche Telekom’s MINDR uses 7+ agents in chain. Allianz Project Nemo uses 7 task-specific agents. Identity must span the chain.

Shadow AI Agent Discovery

80% of organizations have encountered risky AI agent behaviors (McKinsey). Okta and Saviynt now offer shadow agent discovery. 92% of orgs not confident their legacy IAM can handle NHI risks. Sources: McKinsey; Rubrik Zero Labs

MCP Gateway Control

Anthropic’s Model Context Protocol (MCP) is becoming the standard for agent-to-tool communication. Cisco intercepts every MCP request via Duo. Saviynt maps MCP server visibility. OneWelcome needs MCP policy enforcement. Sources: Cisco; Saviynt; Anthropic

Regulatory Convergence

EU AI Act (Aug 2026): biometric ID = high-risk, audit trails mandatory. DORA: real-time access monitoring for 22K entities. NIS2: MFA mandatory across 18 sectors. eIDAS 2.0: wallet by Sep 2026. Five regulations converging simultaneously.
